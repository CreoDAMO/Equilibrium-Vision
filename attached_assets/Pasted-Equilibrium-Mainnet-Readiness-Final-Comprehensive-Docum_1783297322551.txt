6. Signature Enforcement — Live

Component Status Details
Tx signatures ✅ Enforced REQUIRE_TX_SIGNATURES=true — unsigned transactions rejected with 401
Ed25519 batch verification ✅ Live Wired into UTXO multi‑input validation and block assembly — cuts signature‑checking cost during high‑throughput mining
Native WASM multisig ✅ Live Gnosis‑Safe‑style M‑of‑N multisig contract deployed as WASM. Tracks owners by wallet address (never persists public keys on‑chain), uses bitmask for per‑proposal approvals, binds every signed approval to contract address + proposal ID (no replay across proposals/contracts). Replaces single ADMIN_KEY.

Multisig Details

· WAT source compiled at runtime via wabt — single source of truth.
· Host imports added: verify_owner_sig (derives address from public key, compares to stored owner address, verifies Ed25519 signature), self_address (returns contract's own address for replay protection).
· Integration: chain/multisig.ts — deploys and drives the contract; gated validator‑slashing route behind it.
· Fallback: Legacy ADMIN_KEY header only when no multisig configured (backward compatible).

Critical bug caught and fixed: Off‑by‑one in WAT string literal length (29 vs 30 bytes) — caused all signature checks to silently fail. Fixed via hex‑dump isolation and diffing against independently‑computed values.

Verification: 142 TypeScript tests pass (19 new multisig tests covering impersonation, replay, tampered signatures, unknown proposals, threshold logic).
